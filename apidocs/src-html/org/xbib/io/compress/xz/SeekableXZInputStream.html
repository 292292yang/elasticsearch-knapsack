016    /**
017     * Decompresses a .xz file in random access mode.
018     * This supports decompressing concatenated .xz files.
019     * <p/>
020     * Each .xz file consist of one or more Streams. Each Stream consist of zero
021     * or more Blocks. Each Stream contains an Index of Streams' Blocks.
022     * The Indexes from all Streams are loaded in RAM by a constructor of this
023     * class. A typical .xz file has only one Stream, and parsing its Index will
024     * need only three or four seeks.
025     * <p/>
026     * To make random access possible, the data in a .xz file must be splitted
027     * into multiple Blocks of reasonable size. Decompression can only start at
028     * a Block boundary. When seeking to an uncompressed offset that is not at
029     * a Block boundary, decompression starts at the beginning of the Block and
030     * throws away data until the target offset is reached. Thus, smaller Blocks
031     * mean faster seeks to arbitrary uncompressed offsets. On the other hand,
032     * smaller Blocks mean worse compression. So one has to make a compromise
033     * between random access speed and compression ratio.
034     * <p/>
035     * Implementation note: This class uses linear search to locate the correct
036     * Stream from the data structures in RAM. It was the simplest to implement
037     * and should be fine as long as there aren't too many Streams. The correct
038     * Block inside a Stream is located using binary search and thus is fast
039     * even with a huge number of Blocks.
040     * <p/>
041     * <h4>Memory usage</h4>
042     * <p/>
043     * The amount of memory needed for the Indexes is taken into account when
044     * checking the memory usage limit. Each Stream is calculated to need at
045     * least 1&nbsp;KiB of memory and each Block 16 bytes of memory, rounded up
046     * to the next kibibyte. So unless the file has a huge number of Streams or
047     * Blocks, these don't take significant amount of memory.
048     * <p/>
049     * <h4>Creating random-accessible .xz files</h4>
050     * <p/>
051     * When using {@link XZOutputStream}, a new Block can be started by calling
052     * its {@link XZOutputStream#endBlock() endBlock} method. If you know
053     * that the decompressor will need to seek only to certain offsets, it can
054     * be a good idea to start a new Block at (some of) these offsets (and
055     * perhaps only at these offsets to get better compression ratio).
056     * <p/>
057     * liblzma in XZ Utils supports starting a new Block with
058     * <code>LZMA_FULL_FLUSH</code>. XZ Utils 5.1.1alpha added threaded
059     * compression which creates multi-Block .xz files. XZ Utils 5.1.1alpha
060     * also added the option <code>--block-size=SIZE</code> to the xz command
061     * line tool.
062     *
063     * @see SeekableFileInputStream
064     * @see XZInputStream
065     * @see XZOutputStream
066     */
067    public class SeekableXZInputStream extends SeekableInputStream {

151        /**
152         * Creates a new seekable XZ decompressor without a memory usage limit.
153         *
154         * @param in seekable input stream containing one or more
155         *           XZ Streams; the whole input stream is used
156         * @throws XZFormatException           input is not in the XZ format
157         * @throws CorruptedInputException     XZ data is corrupt or truncated
158         * @throws UnsupportedOptionsException XZ headers seem valid but they specify
159         *                                     options not supported by this implementation
160         * @throws java.io.EOFException        less than 6 bytes of input was available
161         *                                     from <code>in</code>, or (unlikely) the size
162         *                                     of the underlying stream got smaller while
163         *                                     this was reading from it
164         * @throws java.io.IOException         may be thrown by <code>in</code>
165         */
166        public SeekableXZInputStream(SeekableInputStream in)
167                throws IOException {
168            this(in, -1);
169        }

171        /**
172         * Creates a new seekable XZ decomporessor with an optional
173         * memory usage limit.
174         *
175         * @param in          seekable input stream containing one or more
176         *                    XZ Streams; the whole input stream is used
177         * @param memoryLimit memory usage limit in kibibytes (KiB)
178         *                    or <code>-1</code> to impose no
179         *                    memory usage limit
180         * @throws XZFormatException           input is not in the XZ format
181         * @throws CorruptedInputException     XZ data is corrupt or truncated
182         * @throws UnsupportedOptionsException XZ headers seem valid but they specify
183         *                                     options not supported by this implementation
184         * @throws MemoryLimitException        decoded XZ Indexes would need more memory
185         *                                     than allowed by the memory usage limit
186         * @throws java.io.EOFException        less than 6 bytes of input was available
187         *                                     from <code>in</code>, or (unlikely) the size
188         *                                     of the underlying stream got smaller while
189         *                                     this was reading from it
190         * @throws java.io.IOException         may be thrown by <code>in</code>
191         */
192        public SeekableXZInputStream(SeekableInputStream in, int memoryLimit)
193                throws IOException {
194            this.in = in;
195            DataInputStream inData = new DataInputStream(in);
196    
197            // Check the magic bytes in the beginning of the file.
198            {
199                in.seek(0);
200                byte[] buf = new byte[XZ.HEADER_MAGIC.length];
201                inData.readFully(buf);
202                if (!Arrays.equals(buf, XZ.HEADER_MAGIC)) {
203                    throw new XZFormatException();
204                }
205            }
206    
207            // Get the file size and verify that it is a multiple of 4 bytes.
208            long pos = in.length();
209            if ((pos & 3) != 0) {
210                throw new CorruptedInputException(
211                        "XZ file size is not a multiple of 4 bytes");
212            }
213    
214            // Parse the headers starting from the end of the file.
215            byte[] buf = new byte[DecoderUtil.STREAM_HEADER_SIZE];
216            long streamPadding = 0;
217    
218            while (pos > 0) {
219                if (pos < DecoderUtil.STREAM_HEADER_SIZE) {
220                    throw new CorruptedInputException();
221                }
222    
223                // Read the potential Stream Footer.
224                in.seek(pos - DecoderUtil.STREAM_HEADER_SIZE);
225                inData.readFully(buf);
226    
227                // Skip Stream Padding four bytes at a time.
228                // Skipping more at once would be faster,
229                // but usually there isn't much Stream Padding.
230                if (buf[8] == 0x00 && buf[9] == 0x00 && buf[10] == 0x00
231                        && buf[11] == 0x00) {
232                    streamPadding += 4;
233                    pos -= 4;
234                    continue;
235                }
236    
237                // It's not Stream Padding. Update pos.
238                pos -= DecoderUtil.STREAM_HEADER_SIZE;
239    
240                // Decode the Stream Footer and check if Backward Size
241                // looks reasonable.
242                StreamFlags streamFooter = DecoderUtil.decodeStreamFooter(buf);
243                if (streamFooter.backwardSize >= pos) {
244                    throw new CorruptedInputException(
245                            "Backward Size in XZ Stream Footer is too big");
246                }
247    
248                // Check that the Check ID is supported. Store it in case this
249                // is the first Stream in the file.
250                check = Check.getInstance(streamFooter.checkType);
251    
252                // Remember which Check IDs have been seen.
253                checkTypes |= 1 << streamFooter.checkType;
254    
255                // Seek to the beginning of the Index.
256                in.seek(pos - streamFooter.backwardSize);
257    
258                // Decode the Index field.
259                try {
260                    index = new IndexDecoder(in, streamFooter, streamPadding,
261                            memoryLimit);
262                } catch (MemoryLimitException e) {
263                    // IndexDecoder doesn't know how much memory we had
264                    // already needed so we need to recreate the exception.
265                    assert memoryLimit >= 0;
266                    throw new MemoryLimitException(
267                            e.getMemoryNeeded() + indexMemoryUsage,
268                            memoryLimit + indexMemoryUsage);
269                }
270    
271                // Update the memory usage and limit counters.
272                indexMemoryUsage += index.getMemoryUsage();
273                if (memoryLimit >= 0) {
274                    memoryLimit -= index.getMemoryUsage();
275                    assert memoryLimit >= 0;
276                }
277    
278                // Remember the uncompressed size of the largest Block.
279                if (largestBlockSize < index.getLargestBlockSize()) {
280                    largestBlockSize = index.getLargestBlockSize();
281                }
282    
283                // Calculate the offset to the beginning of this XZ Stream and
284                // check that it looks sane.
285                long off = index.getStreamSize() - DecoderUtil.STREAM_HEADER_SIZE;
286                if (pos < off) {
287                    throw new CorruptedInputException("XZ Index indicates "
288                            + "too big compressed size for the XZ Stream");
289                }
290    
291                // Seek to the beginning of this Stream.
292                pos -= off;
293                in.seek(pos);
294    
295                // Decode the Stream Header.
296                inData.readFully(buf);
297                StreamFlags streamHeader = DecoderUtil.decodeStreamHeader(buf);
298    
299                // Verify that the Stream Header matches the Stream Footer.
300                if (!DecoderUtil.areStreamFlagsEqual(streamHeader, streamFooter)) {
301                    throw new CorruptedInputException(
302                            "XZ Stream Footer does not match Stream Header");
303                }
304    
305                // Update the total uncompressed size of the file and check that
306                // it doesn't overflow.
307                uncompressedSize += index.getUncompressedSize();
308                if (uncompressedSize < 0) {
309                    throw new UnsupportedOptionsException("XZ file is too big");
310                }
311    
312                // Add this Stream to the list of Streams.
313                streams.add(index);
314    
315                // Reset to be ready to parse the next Stream.
316                streamPadding = 0;
317            }
318    
319            assert pos == 0;
320    
321            // Save it now that indexMemoryUsage has been substracted from it.
322            this.memoryLimit = memoryLimit;
323        }

361        /**
362         * Decompresses the next byte from this input stream.
363         *
364         * @return the next decompressed byte, or <code>-1</code>
365         * to indicate the end of the compressed stream
366         * @throws CorruptedInputException
367         * @throws UnsupportedOptionsException
368         * @throws MemoryLimitException
369         * @throws XZIOException               if the stream has been closed
370         * @throws java.io.IOException         may be thrown by <code>in</code>
371         */
372        public int read() throws IOException {
373            byte[] buf = new byte[1];
374            return read(buf, 0, 1) == -1 ? -1 : (buf[0] & 0xFF);
375        }

377        /**
378         * Decompresses into an array of bytes.
379         * <p/>
380         * If <code>len</code> is zero, no bytes are read and <code>0</code>
381         * is returned. Otherwise this will try to decompress <code>len</code>
382         * bytes of uncompressed data. Less than <code>len</code> bytes may
383         * be read only in the following situations:
384         * <ul>
385         * <li>The end of the compressed data was reached successfully.</li>
386         * <li>An error is detected after at least one but less <code>len</code>
387         * bytes have already been successfully decompressed.
388         * The next call with non-zero <code>len</code> will immediately
389         * throw the pending exception.</li>
390         * <li>An exception is thrown.</li>
391         * </ul>
392         *
393         * @param buf target buffer for uncompressed data
394         * @param off start offset in <code>buf</code>
395         * @param len maximum number of uncompressed bytes to read
396         * @return number of bytes read, or <code>-1</code> to indicate
397         * the end of the compressed stream
398         * @throws CorruptedInputException
399         * @throws UnsupportedOptionsException
400         * @throws MemoryLimitException
401         * @throws XZIOException               if the stream has been closed
402         * @throws java.io.IOException         may be thrown by <code>in</code>
403         */
404        public int read(byte[] buf, int off, int len) throws IOException {
405            if (off < 0 || len < 0 || off + len < 0 || off + len > buf.length) {
406                throw new IndexOutOfBoundsException();
407            }
408    
409            if (len == 0) {
410                return 0;
411            }
412    
413            if (in == null) {
414                throw new XZIOException("Stream closed");
415            }
416    
417            if (exception != null) {
418                throw exception;
419            }
420    
421            int size = 0;
422    
423            try {
424                if (seekNeeded) {
425                    seek();
426                }
427    
428                if (endReached) {
429                    return -1;
430                }
431    
432                while (len > 0) {
433                    if (blockDecoder == null) {
434                        seek();
435                        if (endReached) {
436                            break;
437                        }
438                    }
439    
440                    int ret = blockDecoder.read(buf, off, len);
441    
442                    if (ret > 0) {
443                        curPos += ret;
444                        size += ret;
445                        off += ret;
446                        len -= ret;
447                    } else if (ret == -1) {
448                        blockDecoder = null;
449                    }
450                }
451            } catch (IOException e) {
452                // We know that the file isn't simply truncated because we could
453                // parse the Indexes in the constructor. So convert EOFException
454                // to CorruptedInputException.
455                if (e instanceof EOFException) {
456                    e = new CorruptedInputException();
457                }
458    
459                exception = e;
460                if (size == 0) {
461                    throw e;
462                }
463            }
464    
465            return size;
466        }

468        /**
469         * Returns the number of uncompressed bytes that can be read
470         * without blocking. The value is returned with an assumption
471         * that the compressed input data will be valid. If the compressed
472         * data is corrupt, <code>CorruptedInputException</code> may get
473         * thrown before the number of bytes claimed to be available have
474         * been read from this input stream.
475         *
476         * @return the number of uncompressed bytes that can be read
477         * without blocking
478         */
479        public int available() throws IOException {
480            if (in == null) {
481                throw new XZIOException("Stream closed");
482            }
483    
484            if (exception != null) {
485                throw exception;
486            }
487    
488            if (endReached || seekNeeded || blockDecoder == null) {
489                return 0;
490            }
491    
492            return blockDecoder.available();
493        }

532        /**
533         * Seeks to the specified absolute uncompressed position in the stream.
534         * This only stores the new position, so this function itself is always
535         * very fast. The actual seek is done when <code>read</code> is called
536         * to read at least one byte.
537         * <p/>
538         * Seeking past the end of the stream is possible. In that case
539         * <code>read</code> will return <code>-1</code> to indicate
540         * the end of the stream.
541         *
542         * @param pos new uncompressed read position
543         * @throws XZIOException if <code>pos</code> is negative, or
544         *                       if stream has been closed
545         */
546        public void seek(long pos) throws IOException {
547            if (in == null) {
548                throw new XZIOException("Stream closed");
549            }
550    
551            if (pos < 0) {
552                throw new XZIOException("Negative seek position: " + pos);
553            }
554    
555            seekPos = pos;
556            seekNeeded = true;
557        }
558    
559        /**
560         * Does the actual seeking. This is also called when <code>read</code>
561         * needs a new Block to decode.
562         */
563        private void seek() throws IOException {
564            // If seek(long) wasn't called, we simply need to get the next Block
565            // from the same Stream. If there are no more Blocks in this Stream,
566            // then we behave as if seek(long) had been called.
567            if (!seekNeeded) {
568                if (index.hasNext()) {
569                    BlockInfo info = index.getNext();
570                    initBlockDecoder(info);
571                    return;
572                }
573    
574                seekPos = curPos;
575            }
576    
577            seekNeeded = false;
578    
579            // Check if we are seeking to or past the end of the file.
580            if (seekPos >= uncompressedSize) {
581                curPos = seekPos;
582                blockDecoder = null;
583                endReached = true;
584                return;
585            }
586    
587            endReached = false;
588    
589            // Locate the Stream that contains the uncompressed target position.
590            int i = streams.size();
591            assert i >= 1;
592    
593            long uncompressedSum = 0;
594            long compressedSum = 0;
595    
596            while (true) {
597                index = streams.get(--i);
598                if (uncompressedSum + index.getUncompressedSize() > seekPos) {
599                    break;
600                }
601    
602                uncompressedSum += index.getUncompressedSize();
603                compressedSum += index.getStreamAndPaddingSize();
604                assert (compressedSum & 3) == 0;
605            }
606    
607            // Locate the Block from the Stream that contains
608            // the uncompressed target position.
609            BlockInfo info = index.locate(seekPos - uncompressedSum);
610            assert (info.compressedOffset & 3) == 0 : info.compressedOffset;
611    
612            // Adjust the Stream-specific offsets to file offsets.
613            info.compressedOffset += compressedSum;
614            info.uncompressedOffset += uncompressedSum;
615            assert seekPos >= info.uncompressedOffset;
616            assert seekPos < info.uncompressedOffset + info.uncompressedSize;
617    
618            // Seek in the underlying stream and create a new Block decoder
619            // only if really needed. We can skip it if the current position
620            // is already in the correct Block and the target position hasn't
621            // been decompressed yet.
622            //
623            // NOTE: If curPos points to the beginning of this Block, it's
624            // because it was left there after decompressing an earlier Block.
625            // In that case, decoding of the current Block hasn't been started
626            // yet. (Decoding of a Block won't be started until at least one
627            // byte will also be read from it.)
628            if (!(curPos > info.uncompressedOffset && curPos <= seekPos)) {
629                // Seek to the beginning of the Block.
630                in.seek(info.compressedOffset);
631    
632                // Since it is possible that this Block is from a different
633                // Stream than the previous Block, initialize a new Check.
634                check = Check.getInstance(info.streamFlags.checkType);
635    
636                // Create a new Block decoder.
637                initBlockDecoder(info);
638                curPos = info.uncompressedOffset;
639            }
640    
641            // If the target wasn't at a Block boundary, decompress and throw
642            // away data to reach the target position.
643            if (seekPos > curPos) {
644                // NOTE: The "if" below is there just in case. In this situation,
645                // blockDecoder.skip will always skip the requested amount
646                // or throw an exception.
647                long skipAmount = seekPos - curPos;
648                if (blockDecoder.skip(skipAmount) != skipAmount) {
649                    throw new CorruptedInputException();
650                }
651            }
652    
653            curPos = seekPos;
654        }
655    
656        /**
657         * Initializes a new BlockInputStream. This is a helper function for
658         * <code>seek()</code>.
659         */
660        private void initBlockDecoder(BlockInfo info) throws IOException {
661            try {
662                // Set it to null first so that GC can collect it if memory
663                // runs tight when initializing a new BlockInputStream.
664                blockDecoder = null;
665                blockDecoder = new BlockInputStream(in, check, memoryLimit,
666                        info.unpaddedSize,
667                        info.uncompressedSize);
668            } catch (MemoryLimitException e) {
669                // BlockInputStream doesn't know how much memory we had
670                // already needed so we need to recreate the exception.
671                assert memoryLimit >= 0;
672                throw new MemoryLimitException(
673                        e.getMemoryNeeded() + indexMemoryUsage,
674                        memoryLimit + indexMemoryUsage);
675            } catch (IndexIndicatorException e) {
676                // It cannot be Index so the file must be corrupt.
677                throw new CorruptedInputException();
678            }
679        }
